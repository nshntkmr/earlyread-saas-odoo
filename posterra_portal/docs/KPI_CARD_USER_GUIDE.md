# KPI Card — User Guide

## Overview

KPI Cards display a single key metric with optional trend comparison and visual elements. They are the primary widget type for executive dashboards, command center strips, and at-a-glance performance monitoring.

All KPI card variants are created through the **Dashboard Builder** using the same 5-step wizard. The data comes from SQL queries — either written manually, generated by the AI SQL Assistant, or built with the Visual Builder.

## Variant 1: Stat Card

### What It Shows
- Large formatted value (e.g., "826", "68%", "$1.2M")
- Widget name as label (e.g., "TOTAL ADMITS")
- Trend badge with up/down arrow and percentage change vs prior (e.g., "-12% vs Prior")
- Optional category icon
- Green = improving, Red = declining (configurable via "Lower is Better")

### SQL Requirements

The SQL query must return exactly **2 columns**:

| Column | Purpose | Example |
|--------|---------|---------|
| `value` (x_column) | Current period metric value | `SUM(total_admits)` |
| `prior_value` (y_column) | Prior period metric value | `SUM(CASE WHEN year = %(_year_prior)s ...)` |

The backend auto-computes:
- Formatted value (number, currency, percentage — based on KPI Format setting)
- Percentage change: `(value - prior_value) / abs(prior_value) * 100`
- Trend direction: green arrow up or red arrow down
- Trend text: "+3% vs Prior" or "-12% vs Prior"

### Example SQL (AI-Generated)

**Prompt:** "Total unique patients for the selected year vs. prior year"

```sql
SELECT
    SUM(CASE WHEN year IN %(year)s THEN unique_patients ELSE 0 END) AS value,
    SUM(CASE WHEN year = %(_year_prior)s THEN unique_patients ELSE 0 END) AS prior_value
FROM mv_hha_kpi_summary
WHERE 1=1
    [[AND hha_ccn IN %(hha_ccn)s]]
    [[AND hha_state IN %(hha_state)s]]
    [[AND hha_county IN %(hha_county)s]]
    [[AND hha_city IN %(hha_city)s]]
    [[AND ffs_ma IN %(ffs_ma)s]]
```

### Column Mapping (Step 3)
- **X / Label Column:** `value`
- **Y / Value Column(s):** `prior_value`

### Builder Settings (Step 1)

| Setting | Recommended Value | Notes |
|---------|------------------|-------|
| Chart Type | KPI Card | |
| KPI Style | Stat Card | |
| Lower is Better | Unchecked | Check for metrics like hospitalization rate, mortality |
| Display Mode | Compact or Standard | Compact = smaller card for strips |
| KPI Layout | Inline | Places icon + label on same row |
| Icon | Choose appropriate | People/Users, Hospital, Dollar, etc. |
| Width | 25% for strips, 50% for standalone | |

### Step-by-Step Creation

1. **Step 1 — Chart Type:** Select "KPI Card" → Select "Stat Card" → Set "Lower is Better" if the metric is a rate where decrease = good
2. **Step 2 — Data Source:** Select "AI Assistant" → Pick your Primary Table (e.g., `mv_hha_kpi_summary`) → Type prompt (e.g., "Total admits for the selected year vs. prior year") → Click "Generate SQL" → Review the SQL and column mapping
3. **Step 3 — Columns:** Verify X = `value`, Y = `prior_value`
4. **Step 4 — Filters & Actions:** Configure page, tab, width, height, icon, typography
5. **Step 5 — Preview & Save:** Preview the card, verify data, save

## Variant 2: Stat + Sparkline

### What It Shows
- Everything from Stat Card (value, label, trend badge)
- Plus a **mini trend line** in the top-right corner showing the metric's trajectory across all available time periods
- The sparkline is an ECharts mini line chart — configurable color and fill

### SQL Requirements

The SQL query must return **3 columns**:

| Column | Purpose | Example |
|--------|---------|---------|
| `value` (x_column) | Current period metric value | `SUM(hospitalization_count) / NULLIF(SUM(total_admits), 0) * 100` |
| `prior_value` (y_column) | Prior period metric value | Same formula for prior year |
| `sparkline_data` | CSV string of values across ALL time periods | `STRING_AGG(rate::text, ',' ORDER BY year)` |

The `sparkline_data` column is a comma-separated string of numbers representing the metric value for each time period (e.g., `"9.76,11.2,10.5,8.9"`). The backend parses this into an array and builds an ECharts line chart.

**Critical:** The sparkline CTE must include ALL years — do NOT filter by `%(year)s` in the sparkline computation. Only the `current_year` and `prior_year` CTEs should filter by year.

### Example SQL (AI-Generated)

**Prompt:** "Hospitalization rate for the selected year vs. prior year with a sparkline showing rate trajectory across all available years"

```sql
WITH sparkline AS (
    -- ALL years, no year filter — full trajectory for sparkline
    SELECT
        year,
        SUM(hospitalization_count) / NULLIF(SUM(total_admits), 0) * 100 AS metric_value
    FROM mv_hha_kpi_summary
    WHERE 1=1
        [[AND hha_ccn IN %(hha_ccn)s]]
        [[AND hha_state IN %(hha_state)s]]
        [[AND hha_county IN %(hha_county)s]]
        [[AND hha_city IN %(hha_city)s]]
        [[AND ffs_ma IN %(ffs_ma)s]]
    GROUP BY year
    ORDER BY year
),
current_year AS (
    SELECT
        SUM(hospitalization_count) / NULLIF(SUM(total_admits), 0) * 100 AS value
    FROM mv_hha_kpi_summary
    WHERE 1=1
        [[AND year IN %(year)s]]
        [[AND hha_ccn IN %(hha_ccn)s]]
        [[AND hha_state IN %(hha_state)s]]
        [[AND hha_county IN %(hha_county)s]]
        [[AND hha_city IN %(hha_city)s]]
        [[AND ffs_ma IN %(ffs_ma)s]]
),
prior_year AS (
    SELECT
        SUM(hospitalization_count) / NULLIF(SUM(total_admits), 0) * 100 AS prior_value
    FROM mv_hha_kpi_summary
    WHERE year = %(_year_prior)s
        [[AND hha_ccn IN %(hha_ccn)s]]
        [[AND hha_state IN %(hha_state)s]]
        [[AND hha_county IN %(hha_county)s]]
        [[AND hha_city IN %(hha_city)s]]
        [[AND ffs_ma IN %(ffs_ma)s]]
)
SELECT
    cy.value,
    py.prior_value,
    STRING_AGG(s.metric_value::text, ',' ORDER BY s.year) AS sparkline_data
FROM current_year cy
CROSS JOIN prior_year py
CROSS JOIN sparkline s
GROUP BY cy.value, py.prior_value
```

### Column Mapping (Step 3)
- **X / Label Column:** `value`
- **Y / Value Column(s):** `prior_value, sparkline_data`

### Sparkline-Specific Settings

| Setting | Where | Purpose |
|---------|-------|---------|
| Sparkline Metric Column | Step 2 (AI Assistant) | Optional dropdown — tells AI which numeric column to use for `STRING_AGG`. If empty, AI infers from your prompt. |
| Sparkline Color | Step 1 (Sparkline Settings) | Color of the trend line. Leave empty for palette default. |
| Fill Area Under Line | Step 1 (Sparkline Settings) | Whether to show a filled area below the sparkline. Default: checked. |
| Lower is Better | Step 1 (Trend Settings) | Invert trend colors for metrics where decrease = good. |

### Step-by-Step Creation

1. **Step 1 — Chart Type:** Select "KPI Card" → Select "Stat + Sparkline" → Optionally set sparkline color → Check "Lower is Better" if applicable
2. **Step 2 — Data Source:** Select "AI Assistant" → Pick Primary Table → Optionally select Sparkline Metric Column → Type prompt describing the metric with "sparkline showing trajectory across all years" → Click "Generate SQL"
3. **Step 3 — Columns:** Verify X = `value`, Y = `prior_value, sparkline_data`
4. **Step 4 — Filters & Actions:** Configure page, tab, width, icon
5. **Step 5 — Preview & Save:** Verify the sparkline appears in the top-right corner

## SQL Patterns Reference

### Year Handling

All KPI SQL uses these system-generated helper parameters:

| Parameter | When Single Year Selected | When Multiple Years / "All" |
|-----------|--------------------------|---------------------------|
| `%(year)s` | `('2023',)` | `('2022','2023','2024')` or `('__all__',)` |
| `%(_year_single)s` | `2023` (integer) | `NULL` |
| `%(_year_prior)s` | `2022` (integer) | `NULL` |

### Filter Parameters

| Filter | Parameter | Multi-select | Usage |
|--------|-----------|-------------|-------|
| Provider | `%(hha_ccn)s` | Yes | `IN %(hha_ccn)s` |
| State | `%(hha_state)s` | Yes | `IN %(hha_state)s` |
| County | `%(hha_county)s` | Yes | `IN %(hha_county)s` |
| City | `%(hha_city)s` | Yes | `IN %(hha_city)s` |
| Year | `%(year)s` | Yes | `IN %(year)s` |
| Payer | `%(ffs_ma)s` | No | `= %(ffs_ma)s` |

### `[[...]]` Optional Clauses

Clauses wrapped in `[[...]]` are automatically removed when the parameter is empty or "All":

```sql
WHERE 1=1
    [[AND hha_state IN %(hha_state)s]]   -- removed when State = "All"
    [[AND ffs_ma IN %(ffs_ma)s]]         -- removed when Payer = "All"
```

`WHERE 1=1` is an anchor that ensures valid SQL even when all clauses are removed.

### Rate Computation

For rate metrics (hospitalization rate, timely access, mortality):

```sql
-- CORRECT: Weighted aggregation
SUM(hospitalization_count) / NULLIF(SUM(total_admits), 0) * 100

-- WRONG: Simple average of pre-computed rates
AVG(hospitalization_rate)   -- statistically incorrect!
```

Always use `SUM(numerator) / NULLIF(SUM(denominator), 0)` — never `AVG(rate)`.

## Tips

1. **Use the AI Assistant** — it generates correct SQL with proper rate computations, year handling, and filter clauses
2. **Set "Lower is Better"** for metrics where a decrease is positive (hospitalization rate, mortality, rehospitalization, leakage rate, etc.)
3. **For sparklines**, the trend CTE must NOT filter by year — it needs all years for the full trajectory
4. **Preview before saving** — verify the value, trend direction, and sparkline data look correct
5. **Widget width** — use 25% for command center strips (4 KPIs in a row), 33% for 3-column layouts, 50% for standalone
6. **KPI Format** — set to "Percent" for rates, "Currency" for dollar amounts, "Number" for counts

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| "0% vs Prior" | Y column is wrong (maps to same column as X) | Set Y to `prior_value` only, not `value, prior_value` |
| No trend arrow/color | Widget chart_type doesn't include auto-trend | Ensure chart_type is "KPI Card" (not manually set to something else) |
| Sparkline not showing | SQL filters sparkline data by year (only 1 data point) | Ensure sparkline CTE has NO `[[AND year IN %(year)s]]` filter |
| Sparkline not showing | `sparkline_data` column not in Y columns | Set Y to `prior_value, sparkline_data` |
| Sparkline not showing | `visual_config` missing `kpi_style: sparkline` | Recreate widget through the builder (don't manually create in Odoo admin) |
| Trend colors inverted | "Lower is Better" is checked when it shouldn't be (or vice versa) | Toggle the setting in Step 1 → Trend Settings |
| SQL error: invalid input syntax for smallint | `_year_prior` resolves to empty string when multiple years selected | Fixed in platform — empty helper params auto-convert to NULL |
